# EAGLE Cleanup Inventory

## Scope and evidence

This inventory was created before cleanup edits. The worktree was clean at the
start of the review. The inventory used `git status --short`, `git branch -a`,
`git log --all --oneline`, `git grep`, repository file discovery, Python import
and symbol searches, launcher/configuration tracing, test discovery, and Git
history for deleted paths. The `third_party/microrts/data/` corpus was excluded
from marker searches because it is vendored application data rather than EAGLE
development scaffolding.

## Phase decisions

The first implementation change is limited to duplicate evaluator
consolidation and cleanup documentation. Smoke support is not deleted blindly:
the final-test bounded run has unique real Java/opponent coverage, while the
mock command is a deterministic test workflow that will be renamed in operator
documentation. Historical readers and the LLM endpoint fallback require
schema/config migration evidence before removal. Vendored MicroRTS files and
the TMA adapter remain inside the application boundary.
